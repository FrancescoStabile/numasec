# numasec v1.1.7

This release turns numasec from a strong security agent into a broader operator workbench: more plays, better capability checks, adapter-backed vertical workflows, sharper browser/passive analysis, and a README that finally explains what the project can do.

## Highlights

- Added 11 built-in plays covering web, API, auth, network, AppSec, OSINT, CTF/forensics, cloud posture, container surface, IaC triage, and binary triage.
- Added adapter tools for cloud posture, container images, infrastructure-as-code, and binaries, with honest unavailable/degraded states when local tools are missing.
- Normalized play execution so workflows are easier to test, review, extend, and expose through the `play` tool.
- Improved browser/passive web analysis and runtime readiness checks for a cleaner operator experience.
- Updated AI SDK/provider packages for current provider behavior while keeping the release on the stable Bun 1.3.11 toolchain.
- Added a concise npm postinstall success message so global installs feel intentional instead of silent.
- Rewrote the README around real user-facing capabilities, install paths, operations, plays, adapters, and community extension points.

## Upgrade notes

Install or update with:

```bash
npm install -g numasec
```

The npm package is intentionally small. It installs the wrapper plus the platform-specific binary package, so seeing only a handful of packages added is expected.

For best results, run `/doctor` after upgrading and install any optional local adapters relevant to your workflow: `prowler` for cloud posture, `trivy`/`docker` for containers, `checkov` for IaC, and `checksec` for binary triage.
