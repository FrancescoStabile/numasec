Persistent named secrets (API keys, tokens, passwords) stored locally for reuse across tool calls in the same engagement.

Stored plain-JSON under `~/.config/numasec/secrets.json` (encryption is on the roadmap; keep OS filesystem permissions in mind).

Operations:
- set    — { name, value }            → stores the secret, returns "[REDACTED:<name>]"
- get    — { name }                   → returns the value (use sparingly; only when you must pass it through)
- list   — {}                          → returns the list of known names (never values)
- remove — { name }                   → deletes the secret

Guidelines:
- Prefer passing `[SECRET:<name>]` placeholders in shell commands when possible so the value never lands in the chat transcript; bash will substitute.
- Do not log or echo secret values back to the user.
