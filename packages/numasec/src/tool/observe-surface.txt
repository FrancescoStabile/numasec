Automated attack surface reconnaissance combining port scanning, web crawling, directory fuzzing, and JavaScript analysis into a single composable tool.

Modes:
- recon: TCP port scan followed by service banner probing, plus JS analysis on detected web ports
- crawl: Spider the target to discover URLs, forms, technologies, and linked resources
- dir_fuzz: Brute-force directory and file paths using a wordlist with optional extensions
- js: Analyze JavaScript files for endpoints, SPA routes, API keys, and leaked secrets

Parameters:
- target: Hostname or URL to scan
- modes: Array of modes to run (defaults to all four)
- max_urls: Maximum URLs to collect during crawl (1-500)
- max_depth: Maximum crawl depth (1-6)
- ports: Custom port list for recon mode
- wordlist: Custom wordlist for directory fuzzing
- extensions: File extensions to append during directory fuzzing

Returns a consolidated report with:
- Open ports and identified services with banners
- Discovered URLs and API endpoints
- Detected technologies and frameworks
- HTML forms with their fields and methods
- Leaked secrets (API keys, tokens, credentials) found in JavaScript

Use this tool as the first step in any security assessment to map the target's attack surface before deeper testing with http_request or browser.
