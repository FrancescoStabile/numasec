# /runbook $ARGUMENTS

The operator invoked **/runbook $ARGUMENTS**.

Parse the argument string into:

- an optional action (`list`, `status`, `run`);
- an optional id (runbook id);
- a positional/keyword arg list (everything after the id).

Common invocations:

- `/runbook list`
- `/runbook status web-surface`
- `/runbook web-surface https://example.com`
- `/runbook run web-surface https://example.com`
- `/runbook run appsec-web-triage https://example.com`
- `/runbook run appsec-triage ./src`
- `/runbook run web-surface target=https://x.y domain=x.y`

Rules:

1. If the first token is `list`, call the `runbook` tool with `{ action: "list" }`.
2. If the first token is `status`, call the `runbook` tool with `{ action: "status", id }`.
3. Otherwise default to `action: "run"`:
   - treat the first token as the runbook id unless the first token is literally `run`;
   - bind a single positional arg to the runbook's first required arg (`target`, `path`, etc.);
   - pass `key=value` pairs through as-is.
4. Call the `runbook` tool **once** with the derived `{ action, id, args }`.
5. If the runbook action is `run`, the returned trace becomes the authoritative workflow. Do not reinterpret it, do not call `runbook` again, and do not invent preparatory work outside that trace.
6. Execute the returned trace strictly in order:
   - call exactly one referenced tool per assistant message;
   - wait for that tool result;
   - then immediately call the next pending step;
   - never batch multiple step tools in one response;
   - never stop early while trace steps are still pending.
7. Execute `tool:` lines by calling that tool with the given args. Execute `skill:` lines via the `skill` tool with the named skill and brief.
8. If a referenced tool is not registered or a step fails, say so plainly and continue with the remaining declared steps unless the missing step makes the rest impossible.
9. After all steps complete, produce a short synthesis: what was found, what to do next.
10. Do not invent additional steps or skip declared ones without telling the operator.
11. After the synthesis, stop. Do not call additional tools unless the operator sends a new request.
